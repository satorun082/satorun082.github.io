A問題: I Scream

乳固形分が無脂乳固形分と乳脂肪分の2つから成ることに注意すれば普通に解くことが可能. 自分は,乳固形分=無脂乳固形分だと思いコードを記述してなぜWA(Wrong Answer)食らうのか理解するのに1分ぐらい費やした. 余談だが,氷菓というアニメを見ることをお勧めする.

B問題: Job Assignment

この問題は普通に実直に解くことも可能だったらしい...(前提を考慮しない悪い癖は直さなければいけないが,後悔はしていない) 自身は,O(N)で計算が可能な方法を思いつき,撃沈した. 仕事Aと仕事Bにおける最小値を計算し,max(min(A), min(B))を出力すればokay...ではない. 上記は同じ人が同じ仕事をする場合は,maxではなくsumになってしまうので,その場合は最も早く終わらすことにはならない可能性が出てくる. 結論を言うと,仕事A,Bにおいて,最小の値と二番目に最小の値を保持することにより,問題を解くことが可能である.

C問題: Squared Error

これは,解説を聞いた時に皆頭良きだな-と感じた問題である. 他にも数学的に解く方法があるが,こちらの方が綺麗にコードを記述することが可能なのかも知れない. とにかく,愚直に書くと以下の様な感じになると思う. 

                
                for(int64_t i = 0; i < n; i++){
                    for(int64_t j = 0; j < i; j++){ 
                        ans += (a[i] - a[j])*(a[i] - a[j]); 
                    }
                }
しかしながら,計算量の関係から難しい... ここで考えるのが,jループを変換することが出来ないかということだ. Aの要素が取りうる値は-200から200までである.これを活用しようということだ. iが2の時にjは基本的には1である(ソースコード上では違うが,問題文上での話をしている). iが3の時にはjは1, 2である.この時に,a[j]で計算する値のみを保持していれば,計算も簡単になると言う訳だ. 何を言ってるか分からないだろう...HTMLとCSSで解説を書いてるのだが不慣れ過ぎて色々四苦八苦している点を考慮して,解答コードを見ることをおススメする.

D問題: Journey

これは,正直知ってるか知らないかに大きく依存すると思う. 勿論,知らなくても頑張れば解けるが,ここで覚えとくことを勧める(自身にも言っている). hamayanhamayanさんが言っているこの言葉が一番納得出来るだろう.

「有効なのが来るまでカードを引く期待値は、有効なカードを引く確率の逆数になる。」

終わりに
